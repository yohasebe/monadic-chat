# プライバシーフィルター（個人情報マスキング）

## 概要

プライバシーフィルターは、ユーザーメッセージに含まれる個人を特定可能な情報（PII）を AI プロバイダに送信する前にマスクし、AI の応答に対してマスクされた情報を元の値に復元します。検出処理は別の Docker コンテナ（Microsoft Presidio + spaCy）でローカルに実行され、PII が LLM への往復経路で外部に出ることはありません。

この機能はオプトインで、デフォルトでは無効です。コンテナをインストールしてセッションごとのトグルをオンにするまで、既存のアプリやセッションに影響しません。

## 利用が向く場面

- 実在する氏名・メールアドレス・電話番号を含むメールや文書を作成するとき。
- 個人情報を含むテキストを翻訳するとき。
- 機密性のある識別子を含むスニペットを、手作業で書き換えずに共有したいとき。

## インストール

プライバシーフィルターは独立した Docker コンテナを必要とします。英語ベースラインのイメージは約 1 GB、追加言語ごとに 150-300 MB 増加します。

1. **Settings → Install Options** を開きます。
2. **Privacy Filter (PII Masking)** セクションを探します。
3. **Install Privacy Filter (English baseline)** をチェックします。
4. 任意で追加言語をチェックします: 日本語・ドイツ語・スペイン語・フランス語・イタリア語・オランダ語・ポルトガル語・中国語。
5. **Save** をクリックします。
6. **Settings → Actions** に切り替えて **Build Privacy** をクリックします（メニュー: **Actions → Build Privacy Container** からも実行可能）。マスターチェックがオンかつ Docker が Stopped 状態のときのみ有効化されます。
7. ビルドが完了するまで待ちます。コンソールパネルに進行状況と最終的な「Build of Privacy container has finished」メッセージが表示されます。

アンインストールするには、マスターチェックを外して Save します。実行時にフィルターは無効化されます。イメージ自体を削除するには `docker rmi yohasebe/monadic-privacy` を手動で実行してください。

## セッションごとのトグル

コンテナをインストールすると、プライバシーフィルターをサポートするアプリでは Session Controls パネルに **Privacy Filter** トグルが表示されます。トグルはセッション開始時に必ずオフです。

- 最初のメッセージを送信する前にオンにすると、そのセッションのマスキングが有効になります。
- 最初のメッセージを送ると、トグルはロックされます。**Reset** を押すか別のアプリに切り替えると編集可能な状態に戻ります。
- 最後の選択はアプリごとに記憶されます。あるアプリに戻ると、そのアプリのトグル状態が復元されます。

トグルが無効化されているときは、ホバーすると理由が表示されます:

- *This app does not support Privacy Filter.* — アプリの MDSL に `privacy` ブロックが宣言されていません。
- *Privacy Filter is not installed. Open Settings → Install Options to enable it.* — 環境変数 `PRIVACY_FILTER=true` が設定されていない、もしくはコンテナがまだビルドされていません。

## インジケーターとレジストリ

フィルターがアクティブな場合、チャットヘッダーにインジケーターが表示されます:

- **Privacy ready**（灰色のアンロックアイコン）: フィルターはオンですが、まだエンティティはマスクされていません。
- **Privacy ON (N)**（緑のロックアイコンとカウント）: 復元用のプレースホルダーが現在 N 件登録されています。
- **Privacy error**（赤）: プライバシーコンテナへ到達できませんでした。Docker が起動していてコンテナが正常か確認してください。

インジケーターをクリックすると **Registry Viewer** が開き、各プレースホルダーと元の値、エンティティの種類（PERSON、EMAIL_ADDRESS など）が一覧表示されます。レジストリはセッションメモリ内にのみ保持され、ディスク上の会話ログには書き込まれません。

## マスクされる対象

デフォルトでは次のエンティティタイプをマスクします:

`PERSON`、`EMAIL_ADDRESS`、`PHONE_NUMBER`、`CREDIT_CARD`、`IP_ADDRESS`、`IBAN_CODE`、`US_SSN`、`MEDICAL_LICENSE`、`URL`。

`LOCATION`（「Tokyo」など）や `DATE_TIME`（「Friday」など）といった日常的な表現でノイズを生みやすいタイプはデフォルトから除外されています。

アプリ作者は MDSL でマスク対象を調整できます:

```ruby
privacy do
  enabled true
  mask_types :person, :email, :address  # :address を加えると LOCATION が含まれる
end
```

## 暗号化エクスポート

プライバシーフィルターが現在のセッションで少なくとも 1 件のエンティティをマスクしている場合、**Save** ボタンは 2 つの直交する軸を持つ統合エクスポートダイアログを開きます:

- **Encryption**: AES-256-GCM（Argon2id 鍵導出）でファイルを暗号化します。パスフレーズはユーザーが指定します。
- **Content**: 元の値を復元した内容で、またはプレースホルダーを残した状態で出力します。

代表的な組み合わせは次の 3 つです:

1. **Encrypted + Restored** — 自分以外には開けない個人アーカイブ。
2. **Encrypted + Masked** — 復号後も PII を露出させない解析ログの共有用。
3. **Plain + Masked** — 実名を含まない簡易な共有サンプル。

パスフレーズの要件: 8 文字以上で確認フィールドが一致すること。強度メーターは情報提供のみで、送信を妨げません。

暗号化エクスポートを再度開くには **Load** ボタンを使い、プロンプトに従ってパスフレーズを入力します。レジストリも会話と一緒に復元されるため、続くメッセージで一貫したマスキングが行えます。

## 対応アプリ

次のアプリの MDSL 定義でプライバシーフィルターが有効化されています:

- Mail Composer
- Chat Plus
- Translate
- Second Opinion
- Chat

その他のアプリは `privacy do` を宣言していないため、トグルは無効のままです。コードやメディア、音声生成を主目的とするアプリは除外されています。生成されたコード内にプレースホルダーが残るなど、本来の出力にマスキングが干渉してしまうためです。

## 制限事項

- アシスタント側の履歴は再マスクされません。パイプラインは各ターンの user ロールに対して動作するため、アシスタントが後のターンで同じ氏名を再利用すると、その箇所はマスクされずに通過します。
- カスタム認識器はプライバシーコンテナのソース（`docker/services/privacy/recognizers/`）に存在します。新しい認識器を追加するにはコンテナの再ビルドが必要です。
- マスキングはメッセージごとにわずかなレイテンシを追加します。テキスト長と有効化された言語によりますが、通常 50-200 ms 程度です。
- カバレッジと精度は言語によって異なります。各 spaCy モデルは異なるコーパスで訓練されているためです。英語が最も検証が進んでおり、その他の言語は追加の spaCy モデルとデフォルトの Presidio 認識器によって対応しています。

## 設定リファレンス

`~/monadic/config/env` に保存される設定:

| 変数 | 値 | デフォルト | 説明 |
|---|---|---|---|
| `PRIVACY_FILTER` | `true` / `false` | `false` | マスタゲート。プライバシーコンテナのビルドおよび利用には `true` が必須です。 |
| `PRIVACY_LANGS` | カンマ区切りの言語コード | `en` | インストールする spaCy 言語モデル（例: `en,ja,de`）。英語は常に含まれます。 |
| `PRIVACY_DEV_PORT` | ポート番号 | `8001` | 開発モード時のみ、プライバシーコンテナの HTTP ポートをホストに公開する用途で使われます。 |

`PRIVACY_FILTER` または `PRIVACY_LANGS` を変更した後は、コンテナを再ビルド（Settings → Actions → Build Privacy）し、Monadic Chat を再起動して新しい環境変数を反映させてください。
