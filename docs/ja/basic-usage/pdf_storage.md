# PDFナレッジベース

Monadic Chat はローカル PDF ナレッジベースを構築でき、AI エージェントが会話中にその内容を検索します。ドキュメントの処理はすべてローカルで完結します。各 PDF からテキストを抽出し、ローカルの sentence-transformer コンテナで埋め込みベクトルに変換し、Qdrant ベクトルデータベースに保存します。保存・検索のいずれにも外部 API キーは不要です。

## 仕組み

- PDF はローカルの `multilingual-e5-base` でテキスト埋め込みに変換されます
- ベクトルとメタデータは Qdrant Docker ボリュームに永続化されます
- 検索はローカルで生成された埋め込みに対する Qdrant の HNSW インデックスを使用します
- オフラインで動作し、保存・検索のいずれにも外部 API キーが不要です

## PDFコレクションの管理

Web インターフェースの PDF データベースパネルでインポート済みドキュメントを管理できます。

- **インポート**: 新しい PDF をアップロード。バックグラウンドでチャンク化・埋め込み・インデックス登録が行われます
- **更新**: 最近追加されたドキュメントを表示するためにリストを更新します
- **削除（項目ごと）**: 確認付きで個別の PDF を削除します
- **すべてクリア**: 現在のアプリ名前空間にあるすべての PDF を確認付きで削除します

各アプリは Qdrant コレクション内で独自の名前空間（`app_key` ペイロードフィールド）を持つため、あるアプリでインポートした PDF が別のアプリに混入することはありません。

## 注意事項

- **データの永続化**: ドキュメントは `monadic-chat-qdrant-data` Docker ボリュームに保存されます
- **バックアップ**: Electron メニューのドキュメントデータベースのエクスポート機能で `.tar.gz` スナップショットを作成できます
- **コンテナの再構築**: データ損失を防ぐため、コンテナを再構築する前にデータベースをエクスポートしてください

## 旧バージョンからのアップグレード

1.0.0-beta.13 以前からアップグレードする場合、旧ストレージ形式は現在のローカルスタックと互換性がありません：

- 既存のローカル PDF は自動移行されません — アップグレード後に PDF を再インポートしてください
- 再インポート完了後、レガシーの `monadic-chat-pgvector-data` Docker ボリュームは削除できます
- クラウド PDF ストレージ（OpenAI Vector Store）は 1.0.0-beta.16 以降サポートされません。OpenAI Files / Vector Store にアップロード済みの PDF はアップグレード後に Monadic Chat からアクセスできなくなります。データは OpenAI アカウント側に残るため、OpenAI ダッシュボードから手動で削除できます。
