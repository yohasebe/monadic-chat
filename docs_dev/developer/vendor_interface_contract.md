# Vendor Helper Interface Contract

This document defines the interface contract that vendor helper modules must implement to integrate with Monadic Chat's provider system.

## Overview

Vendor helpers are Ruby modules located in `lib/monadic/adapters/vendors/`. Each module is included into `MonadicApp` subclasses via the DSL's `include_modules` mechanism. The system expects certain methods to exist on helpers for model discovery, querying, streaming, and tool execution.

## Base Infrastructure

**`BaseVendorHelper`** (`lib/monadic/adapters/base_vendor_helper.rb`) provides shared macros:

| Macro | Purpose | Generated Methods |
|---|---|---|
| `define_timeouts(prefix, open:, read:, write:)` | Timeout configuration from CONFIG | `open_timeout`, `read_timeout`, `write_timeout` |
| `define_models_cache(cache_key)` | Model cache clearing | `clear_models_cache` |
| `retry_with_backoff(max_retries:, delay:)` | Generic retry wrapper | (instance method) |

## Required Interface Methods

### 1. `vendor_name` (class method)

Returns the human-readable provider name for logging and display.

```ruby
def self.vendor_name
  "OpenAI"
end
```

**Implementation status:**

| Provider | Implemented | Scope |
|---|---|---|
| OpenAI | ✓ | class method (in `self.` block) |
| Claude | ✓ | class method (in `self.` block) |
| Gemini | ✓ | `def self.vendor_name` |
| Cohere | ✓ | class method (in `self.` block) |
| Mistral | ✓ | class method (in `self.` block) |
| DeepSeek | ✓ | class method (in `self.` block) |
| Grok | ✓ | class method (in `self.` block) |
| Perplexity | ✓ | `def self.vendor_name` |

### 2. `list_models` (class method)

Fetches available models from the provider API. Results are cached in the global `$MODELS[provider_key]` hash.

```ruby

  # Return Array of model ID strings
  # Return empty array on failure (graceful degradation)
end
```

### 5. `api_request(role, session, call_depth: 0, &block)` (instance method)

Core streaming handler. Called from the WebSocket streaming loop to process user messages and stream responses back to the client.

```ruby
def api_request(role, session, call_depth: 0, &block)
  # role - "user" or "assistant"
  # session - Rack session hash with :messages, :parameters, etc.
  # call_depth - recursion depth for tool call chains (max: settings[:call_depth_limit])
  # block - callback yielded with streaming fragments
  # Returns: Array [text_content, hash_data_or_nil, tool_results_or_nil]
end
```

**Called from:** `lib/monadic/utils/websocket/streaming_handler.rb`

**Implementation status:**

| Provider | Implemented | Extra Params |
|---|---|---|
| OpenAI | ✓ | — |
| Claude | ✓ | — |
| Gemini | ✓ | — |
| Cohere | ✓ | — |
| Mistral | ✓ | — |

### 6. `process_functions(app, session, tools, context, call_depth, &block)` (instance method)

Executes tool/function calls returned by the LLM and feeds results back for continued generation.

```ruby
def process_functions(app, session, tools, context, call_depth, &block)
  # app - MonadicApp instance
  # session - Rack session hash
  # tools - Array of tool call objects (provider-specific format)
  # context - conversation context hash (may be nil)
  # call_depth - current recursion depth
  # block - streaming callback
  # Returns: Array [text_content, hash_data_or_nil]
end
```

**Called from:** `api_request` when the LLM returns tool calls

**Implementation status:**

| Provider | Implemented | 3rd Param Name | Has `context` Param |
|---|---|---|---|
| OpenAI | ✓ | `tools` | ✓ |
| Claude | ✓ | `tools` | ✓ |
| Gemini | ✓ | `tool_calls` | ✓ |
| Cohere | ✓ | `tool_calls` | ✓ |
| Mistral | ✗ | **Inline in api_request** | — |
| DeepSeek | ✓ | `tools` | ✓ |
| Grok | ✓ | `tools` | ✓ |
| Perplexity | ✓ | `tools` | ✓ |
| Ollama | ✓ | `tool_calls` | ✓ |
| Tavily | ✗ | N/A | — |

**Gap:**
- **Mistral** handles tool execution inline within `api_request` — no separate `process_functions` method

## Interface Consistency Summary

| Method | 9-provider Coverage | Gaps |
|---|---|---|
| `vendor_name` | 9/9 (100%) | — |
| `list_models` | 9/9 (100%) | — |
| `clear_models_cache` | 9/9 (100%) | — |
| `send_query` | 9/9 (100%) | — |
| `api_request` | 9/9 (100%) | Grok has extra param (non-breaking) |
| `process_functions` | 8/9 (89%) | Mistral (inline) |

**Note:** Tavily is excluded as a search-only provider with a single `tavily_search` method.

## Parameter Naming Inconsistency

The 3rd parameter of `process_functions` uses two naming conventions:
- **`tools`**: OpenAI, Claude, DeepSeek, Grok, Perplexity (5 providers)
- **`tool_calls`**: Gemini, Cohere, Ollama (3 providers)

This does not affect functionality (positional parameter), but creates inconsistency when reading code.

## Registration in DSL

Each vendor must be registered in `ProviderConfig::PROVIDER_INFO` (`lib/monadic/dsl.rb`):

```ruby
PROVIDER_INFO = {
  "openai" => {
    helper_module: "OpenAIHelper",
    api_key: "OPENAI_API_KEY",
    default_model_env: "OPENAI_DEFAULT_MODEL",
    display_group: "OpenAI",
    aliases: ["open_ai"]
  },
  # ...
}
```

Required fields: `helper_module`, `api_key` (nil for keyless providers like Ollama), `default_model_env`, `display_group`, `aliases`.

## Adding a New Provider

Checklist:

1. Create `lib/monadic/adapters/vendors/new_provider_helper.rb`
2. `include BaseVendorHelper`
3. Call `define_timeouts` and `define_models_cache`
4. Implement all 6 contract methods (vendor_name, list_models, clear_models_cache, send_query, api_request, process_functions)
5. Use `tools` (not `tool_calls`) for `process_functions` 3rd parameter
6. Include `context` as 4th parameter of `process_functions`
7. Register in `ProviderConfig::PROVIDER_INFO`
8. Add model specifications to `model_spec.js`
9. Add API key mapping in `monadic.rb`
10. Update frontend `getProviderFromGroup()` in `monadic.js`
11. Add tests

See also: CLAUDE.md "Adding a New AI Provider" checklist.
